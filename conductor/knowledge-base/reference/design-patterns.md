## TL;DR

Design patterns are reusable solutions to common software design problems. They fall into Creational (how to create objects), Structural (how to compose objects), and Behavioral (how objects communicate). For interviews: know Singleton, Factory, Repository, Observer, and Strategy patterns.

## Interview Q&A

### L1: What is the Singleton pattern?
**Answer:** Ensures a class has only one instance and provides a global access point. Used for logging, configuration, connection pools.

### L2: Explain the Repository pattern
**Answer:** Abstracts data access, providing a collection-like interface. Decouples business logic from persistence details.
